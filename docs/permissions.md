# Permissions

## SUPER_ADMIN
Can create tenants, invite users, approve and publish versions, configure domains,
adjust credits, manage integrations, and view all tenants.

## CLIENT_ADMIN
Can access only assigned tenants and permitted dashboard features.

## CLIENT_VIEWER
Read-only access to permitted leads, analytics, and reports.

Authorization must be enforced in the API and database. Frontend route protection alone is insufficient.
